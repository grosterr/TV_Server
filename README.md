# TV_Server

Локальний медіасервер, розроблений для забезпечення потокового відтворення торентів та агрегованого пошуку. Ідеально підходить для використання з додатком **Lampa** на Smart TV, Android TV, або ПК.

## 🚀 Архітектура та склад

Цей стек розгорнуто в ізольованому середовищі за допомогою **Docker Compose** і складається з двох основних компонентів:

1. **TorrServer** (порт `8090`)
   - **Призначення:** Потокове відтворення торентів (Streaming) без необхідності попереднього завантаження всього файлу на жорсткий диск.
   - **Оптимізація:** Скрипт `configure.ps1` вмикає розширений кеш в оперативній пам'яті (2 ГБ) та збільшує ліміт P2P-з'єднань (до 1000) для максимальної утилізації ширини каналу (до 300+ Мбіт/с). Значення налаштовуються у файлі `.env`.
   - Дані кешуються виключно в ОЗП і видаляються після перегляду.

2. **Jackett** (порт `9117`)
   - **Призначення:** Уніфікований проксі-сервер для пошуку по торент-трекерам. Перекладає пошукові запити від Lampa у запити до конкретних сайтів.
   - **Трекери:** Підтримує додавання будь-яких відкритих або закритих трекерів (наприклад: Toloka.to, ThePirateBay, 1337x).

## 🛠 Як запустити (Встановлення)

1. **Підготовка:**
   Впевніться, що у вас встановлено [Docker Desktop](https://www.docker.com/products/docker-desktop/) (якщо ви на Windows) або Docker + Docker Compose (якщо на Linux).
   **Важливо для автозапуску:** У налаштуваннях Docker Desktop (Settings -> General) увімкніть галочку **"Start Docker Desktop when you log in"**. Завдяки параметру `restart: unless-stopped`, сервер буде автоматично запускатися разом із ПК як фонова служба.

2. **Запуск контейнерів:**
   Відкрийте термінал, перейдіть в папку `media-server` і виконайте команду:
   ```bash
   docker compose up -d
   ```

3. **Базове налаштування Jackett:**
   - Відкрийте браузер і перейдіть за адресою: `http://localhost:9117`
   - У верхньому правому кутку скопіюйте ваш **API Key**. Він знадобиться для підключення до Lampa.

4. **Автоматичне налаштування (рекомендовано):**
   Замість того, щоб додавати кожен трекер вручну, скористайтеся скриптом. Він налаштує оптимізацію TorrServer і додасть усі приватні трекери за один крок.
   - Скопіюйте шаблон і впишіть свої дані:
     ```powershell
     Copy-Item .env.example .env   # потім відредагуйте .env
     ```
   - У `.env` вкажіть `JACKETT_APIKEY` (з веб-інтерфейсу Jackett) та логіни/паролі трекерів.
   - Запустіть:
     ```powershell
     ./configure.ps1
     ```
   Скрипт **ідемпотентний** — його можна безпечно запускати повторно. Щоб додати новий приватний трекер, достатньо дописати один рядок у масив `$Trackers` всередині `configure.ps1` та відповідні змінні у `.env`.

> 🔒 **Безпека:** файл `.env` містить паролі й API-ключ і **не потрапляє в git** (додано в `.gitignore`). Ніколи не комітьте його. Публічні трекери (ThePirateBay, 1337x) не потребують логіну — їх можна додати кнопкою **"+ Add indexer"** у веб-інтерфейсі Jackett.

## 📺 Підключення до Lampa

Для того, щоб Lampa побачила ваш новий сервер і почала шукати та відтворювати фільми, виконайте наступні кроки:

### Варіант А: Smart TV (Samsung Tizen) через Media Station X
Якщо у вас телевізор Samsung, найзручніший спосіб запустити Lampa — через безкоштовний додаток-лаунчер:
1. Завантажте **Media Station X** з офіційного магазину Samsung Apps.
2. Запустіть його ➔ **Settings** ➔ **Start Parameter** ➔ **Setup**.
3. Введіть `lampa.mx` та підтвердіть. Додаток перезапуститься і відкриє Lampa.

### Варіант Б: Налаштування самої Lampa (усі пристрої)
1. **TorrServer:** Перейдіть у **Налаштування** -> **Торренти**.
   - Увімкніть використання TorrServer.
   - Вкажіть **Основне посилання**: `http://[IP-АДРЕСА-ВАШОГО-ПК]:8090` (наприклад, `http://192.168.1.165:8090`).
2. **Jackett:** Перейдіть у **Налаштування** -> **Плагіни**.
   - Перейдіть у налаштування плагіна Jackett і вкажіть:
   - **Посилання:** `http://[IP-АДРЕСА-ВАШОГО-ПК]:9117`
   - **API ключ:** Вставте скопійований ключ з вебінтерфейсу Jackett.

## 💡 Часті питання (FAQ) та вирішення проблем

### 1. Немає звуку під час перегляду (BDRip / Blu-Ray)
- **Причина:** Великі торенти в якості BDRip/Remux використовують багатоканальні аудіоформати (**AC3 / DTS**). Вбудовані плеєри в браузерах, WebOS та Tizen часто не мають ліцензії на їх відтворення, тому відео йде без звуку.
- **Рішення:** 
  - На ПК: Використовуйте зовнішній плеєр (VLC, PotPlayer), вибравши у Lampa опцію "Відкрити у VLC".
  - На Smart TV (Android): Встановіть Vimu Media Player, MX Player або VLC і виберіть його в налаштуваннях Lampa (Налаштування -> Плеєр). 
  - На телевізорах Samsung/LG: Обирайте для перегляду торенти формату **WEB-DL** / **WEBRip** (вони зазвичай мають сумісний звук формату AAC).

### 2. Чому швидкість відображається як 30 МБ/с, якщо у мене тариф 300 Мбіт/с?
- Зверніть увагу на одиниці виміру: провайдери вказують швидкість у Мега**бітах** (Мбіт/с), а Lampa чи TorrServer часто показують у Мега**байтах** (МБ/с, МБайт/с). 
- Оскільки 1 Байт = 8 біт, швидкість 33 МБ/с насправді дорівнює ~265 Мбіт/с. Це означає, що програма працює на максимумі!

### 3. Швидкість завантаження не досягає максимуму, а "стрибає"
- Торент (BitTorrent) — це P2P-мережа. Швидкість залежить від кількості звичайних людей (сідерів), які зараз роздають цей файл, та їхньої швидкості віддачі. 
- Також TorrServer "розумно" знижує швидкість, коли кеш заповнюється, щоб не перевантажувати комп'ютер. Фільм не завантажується повністю, а лише буферизується на льоту.

### 4. Що буде з фільмом, якщо перезавантажити комп'ютер?
- Якщо ви перезавантажите ПК (на якому працює сервер) під час перегляду, фільм на телевізорі зависне. Оскільки TorrServer кешує відео тільки в оперативній пам'яті (щоб не псувати жорсткий диск), після рестарту ПК кеш очиститься.
- **Як продовжити:** Просто увімкніть фільм на телевізорі знову і перемотайте на потрібну хвилину. TorrServer миттєво підключиться до сідерів і продовжить завантаження саме з того моменту (зазвичай це займає 3-5 секунд).

### 5. Lampa на телевізорі пише «парсер не відповідає», хоча з телефону сторінка `:9117` відкривається
Це майже завжди одна з двох причин (сторінка відкривається з телефону, бо це звичайна навігація браузера, а Lampa робить фонові JS-запити, до яких застосовуються інші правила безпеки):

1. **CORS вимкнено в Jackett.** Браузер блокує відповідь Jackett, бо запит іде з чужого домену (`lampa.mx`).
   - **Рішення:** у файлі `jackett_config/Jackett/ServerConfig.json` встановіть `"AllowCORS": true` і перезапустіть контейнер: `docker restart jackett`.
2. **Mixed content (HTTPS → HTTP).** Якщо Lampa відкрита по `https://`, а Jackett працює по `http://`, браузер телевізора блокує «незахищений» запит.
   - **Рішення:** у Media Station X у полі *Start Parameter* вкажіть Lampa через **http** (`http://lampa.mx`), щоб і Lampa, і Jackett були по HTTP.

Також перевірте, що:
- В Lampa вказано саме LAN-IP ПК, а не `localhost`: `http://192.168.1.165:9117` (свій IP: `ipconfig` → рядок IPv4).
- Телевізор і ПК в **одній мережі** (без «гостьового» Wi-Fi чи AP-isolation).
- Брандмауер Windows пропускає вхідні на порт `9117` для приватної мережі.
