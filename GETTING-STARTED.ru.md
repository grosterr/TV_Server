Гайд разделён на две части:

- **[ЧАСТЬ 2 — Как сервер для ТВ и Android](#часть-2--как-сервер-для-тв-и-android)** — продолжение. Делаете из компьютера домашний сервер, чтобы смотреть на телевизоре, приставке или телефоне.

> ⏱ Часть 1 — ~15 минут. Часть 2 — ещё ~15 минут. Технических знаний не нужно: уметь устанавливать программы и копировать текст.

## 📖 Что это вообще такое (простыми словами)

Представьте это как **собственный Netflix, который показывает фильмы из торрентов**, но без загрузки и без рекламы. На компьютере крутится маленький «движок», который ищет и воспроизводит видео на лету — ничего не сохраняется на диск.

Эта сборка называется **Torlamp** и состоит из трёх частей:
- **TorrServer** — «движок», превращающий торрент в видеопоток.
- **Jackett** — «поисковик» по торрент-сайтам.
- **FlareSolverr** — помощник, обходящий защиту Cloudflare на трекерах (чтобы работало больше сайтов). Опциональный: инсталлятор спросит, включать ли его (по умолчанию — да); на слабых устройствах можно выключить и сэкономить ~0.5 ГБ RAM.

Управляете всем через приложение **Lampa** — красивый каталог с постерами.

> ℹ️ **Важно про Lampa (из официального FAQ разработчиков):** Lampa — это не готовое решение, а **«конструктор» (skeleton)**, который нужно **самостоятельно настроить** (указать ваш TorrServer и парсер — именно это мы и делаем ниже). Это отдельное бесплатное приложение, доступное нативно почти везде:
> - **ПК:** [Lampa Desktop](https://github.com/Kolovatoff/lampa-desktop/releases) (Windows/Mac/Linux)
> - **Android / Android TV / приставки:** [официальные релизы lampa-app/LAMPA](https://github.com/lampa-app/LAMPA/releases)
> - **Веб / Smart TV:** сайт [lampa.mx](https://lampa.mx)
> - **Исходники / самохостинг:** [yumata/lampa](https://github.com/yumata/lampa)

# ЧАСТЬ 1 — Использование на ПК

> 🎯 Цель этой части: установить всё и смотреть фильмы **на этом же компьютере**. Без сетевых настроек — всё через адрес `localhost` (то есть «этот компьютер»).

> ⚡ **Самый лёгкий путь (рекомендуется):** сделайте только **Шаг 1** (Docker) и **Шаг 2** (загрузка), а затем запустите установщик — он выполнит Шаги 3–4 за вас:
> - **Windows:** дважды кликните **`install.bat`** в корне проекта.
> - **Linux / Mac:** из корня проекта — `bash install.sh`
>
> Установщик спросит **язык** (English / Українська / Русский), при желании включит скрытие IP (WARP), поднимет сервер, настроит Jackett, **сам подхватит API-ключ** и **откроет веб-интерфейс Jackett**, где вы добавите индексаторы. Все готовые адреса и ключ он сохранит в файл **`lampa_settings.txt`** (в корне проекта) — оттуда возьмёте их для Lampa (Шаг 5). Если сервер уже стоит, будет выбор **REPAIR / DELETE / QUIT**, а когда на GitHub вышла более новая версия — ещё и **UPDATE**: установщик сам скачает новые файлы (ваши данные и настройки не трогает) и обновит Docker-образы. Хотите всё вручную — Шаги 3–4 ниже.

## Шаг 1. Установить Docker Desktop

«Docker» — программа, запускающая наш медиасервер в изолированном «контейнере». Устанавливается как обычное приложение.

1. Откройте [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) → **Download for Windows**.
2. Запустите файл, жмите **OK / Next** до конца, разрешите перезагрузку при необходимости.
3. Запустите **Docker Desktop** (иконка кита 🐳). Дождитесь, пока кит внизу слева станет зелёным — «готов».

> ⚠️ Если Docker попросит починить «WSL 2» — он сам покажет кнопку. Нажмите её и перезагрузитесь.

## Шаг 2. Скачать этот проект

1. Откройте страницу **[Releases](../../releases/latest)** проекта и скачайте готовую сборку **`torlamp-v1.0.zip`** (в разделе *Assets*) — это «чистый» архив без служебных файлов.
   > Как альтернатива — зелёная кнопка **`< > Code` → Download ZIP** (весь репозиторий вместе со служебными файлами).
2. Распакуйте, например, в `C:\Torlamp`.

## Шаг 3. Первый запуск

1. Откройте подпапку **`media-server`**.
2. В пустом месте внутри папки: **Shift + правая кнопка мыши** → **«Открыть окно PowerShell здесь»**.
3. Введите и нажмите **Enter**:
   ```powershell
   docker compose up -d
   ```
4. Первый раз несколько минут загружаются компоненты. Когда увидите строки `Started` — сервер работает. 🎉

**Проверка:** откройте [http://localhost:9117](http://localhost:9117) — должен открыться Jackett.

## Шаг 4. Настроить поиск (Jackett)

Здесь вы говорите серверу, по каким сайтам искать фильмы. Всё делается на панели Jackett: **[http://localhost:9117/UI/Dashboard](http://localhost:9117/UI/Dashboard)**

1. **Скопируйте API Key** — длинную строку в правом верхнем углу. Она понадобится дальше.
2. **Добавить трекер:** кнопка **`+ Add indexer`** → найдите нужный (например `Toloka`, `The Pirate Bay`, `1337x`) → нажмите жёлтый **`+`**.
   - Публичные (PirateBay, 1337x) добавляются сразу.
   - Приватные (Toloka) попросят ваш **логин и пароль** от того сайта.
   - 💪 Благодаря встроенному FlareSolverr работают даже трекеры с защитой Cloudflare (1337x и т. п.).
3. **Удалить/изменить трекер:** на той же панели рядом с каждым добавленным трекером есть значки — корзина (удалить), гаечный ключ (настроить), лупа (проверить).

> 💡 Оптимизацию сервера (кеш TorrServer, CORS, подключение FlareSolverr) можно применить скриптом `configure.ps1` — см. [технический README](README.ru.md). А сами трекеры добавляются здесь же, в веб-интерфейсе Jackett.

## Шаг 5. Смотреть на ПК 🍿

**Рекомендуемый способ — нативное приложение [Lampa Desktop](https://github.com/Kolovatoff/lampa-desktop/releases)** (Windows/Mac/Linux). Оно надёжнее браузера: помнит настройки, не имеет проблем с блокировкой «незащищённых» запросов и легко подключает внешние плееры.

1. Скачайте и установите Lampa Desktop из [релизов](https://github.com/Kolovatoff/lampa-desktop/releases).
2. В **Настройках** укажите два «локальных» адреса:
   - **TorrServer** (Настройки → Торренты): `http://localhost:8090`
   - **Jackett / Парсер** (Настройки → Парсер): ссылка `http://localhost:9117` + **API Key** из Шага 4.
3. Найдите фильм → **«Смотреть»** → выберите раздачу (где больше **сидеров** — тем быстрее) → воспроизведение через несколько секунд.

> 💡 **Быстрая альтернатива без установки:** откройте [lampa.mx](https://lampa.mx) прямо в браузере и укажите те же адреса. Работает, но браузер может блокировать запросы к `localhost` с HTTPS-сайта — тогда либо согласитесь на «небезопасный контент» в настройках сайта, либо воспользуйтесь десктоп-приложением выше.
>
> Если видео не играет из-за формата — откройте поток во **VLC** («Открыть во внешнем плеере») или через веб-интерфейс TorrServer: [http://localhost:8090](http://localhost:8090).

**Готово! На компьютере всё работает.** Если этого достаточно — можно остановиться здесь. Хотите смотреть на телевизоре — переходите к Части 2. 👇

# ЧАСТЬ 2 — Как сервер для ТВ и Android

> 🎯 Цель: превратить компьютер в **домашний сервер**, чтобы смотреть фильмы на телевизоре, приставке или телефоне. Компьютер делает всю работу, остальные устройства только показывают.

**Что меняется по сравнению с Частью 1:** вместо `localhost` теперь нужно указывать **реальный адрес компьютера в сети**, а также разрешить соединения в брандмауэре.

### ⚠️ Главное условие
Компьютер и телевизор должны быть **в одной домашней сети** (один роутер/Wi-Fi). И компьютер должен быть **включён во время просмотра**.

---

## Шаг 6. Узнать адрес компьютера

Если вы ставили через **установщик**, он уже спросил, **какой сетевой адаптер использовать**, и сохранил готовый адрес вместе с API-ключом в файл **`lampa_settings.txt`** (в корне проекта) — просто откройте его и берите значения оттуда. Тогда Шаг 6 можно пропустить.

Иначе узнайте адрес вручную:
1. В окне PowerShell введите:
   ```powershell
   ipconfig
   ```
2. Найдите строку **«IPv4-адрес»** вашей домашней сети — что-то вроде `192.168.1.165` или `10.0.0.5`.
   > ⚠️ Если адресов несколько — **игнорируйте виртуальные адаптеры** (WSL, Hyper-V, vEthernet, VirtualBox). Берите тот, что начинается с `192.168.` или `10.` и относится к вашему Wi-Fi/Ethernet. Именно поэтому установщик и спрашивает, какой адаптер выбрать.
3. Запишите это число. **Это адрес вашего сервера** для всех устройств.

> 💡 Чтобы адрес не менялся после перезагрузки, в настройках роутера «закрепите» его за компьютером (Static DHCP / резервирование). Не обязательно, но удобно.

## Шаг 7. Разрешить соединения в брандмауэре

Windows по умолчанию может блокировать входящие подключения с других устройств. Разрешите порты сервера — в PowerShell **от имени администратора**:

```powershell
New-NetFirewallRule -DisplayName "MediaServer TorrServer" -Direction Inbound -LocalPort 8090 -Protocol TCP -Action Allow
New-NetFirewallRule -DisplayName "MediaServer Jackett"    -Direction Inbound -LocalPort 9117 -Protocol TCP -Action Allow
```

> 💡 Этих двух портов достаточно для просмотра в домашней сети. Порт **42116** (входящие P2P-соединения) открывать здесь **не обязательно**: без него стриминг работает, но входящие пиры ограничены, поэтому скорость ниже. Хотите выжать максимум (и пробросить порт на роутере для доступа из интернета) — см. раздел «Оптимизация скорости на уровне P2P» в [README](README.ru.md).

> Как открыть PowerShell администратором: меню Пуск → напишите «PowerShell» → правой кнопкой → «Запуск от имени администратора».

## Шаг 8. Установить Lampa на телевизор

### 🤖 Android TV / приставка (рекомендуемый способ — через Downloader)
Удобнее всего — приложение **Downloader**, которое по одному коду ставит сразу Lampa и плеер Vimu. ([Официальный гайд разработчиков Lampa](https://telegra.ph/Ustanovka-prilozhenij-Lampa-Torrserve-Matrix-Vimu-na-Android-TV-s-pomoshchyu-Downloader-11-18))

1. **Установите Downloader.** В **Google Play** на телевизоре найдите **Downloader by AFTVnews** (оранжевый значок со стрелкой) → **Установить**.
2. **Откройте нужную страницу.** Запустите Downloader → в поле кода введите:
   ```
   4384169
   ```
   → нажмите **Go / OK**. Откроется страница «Загрузка APK».
3. **Выберите приложение** из списка и нажмите **OK → Download**:
   - 📺 **Lampa** — обязательно (само приложение)
   - ▶️ **Vimu Media Player** — советую (внешний плеер для DTS/4K, решает проблемы со звуком)
   - 🧲 Torrserve MatriX — **не нужен**: TorrServer уже работает на вашем ПК (см. ниже).
4. После загрузки нажмите **Install**.
5. **Если ТВ заблокирует установку** («Ваш телевизор блокирует неизвестные приложения…») → **Настройки** → разрешите установку для **Downloader** → вернитесь и нажмите Install ещё раз.

> 💡 **Не устанавливайте TorrServer на телевизор.** Ваш сервер уже крутится на ПК — телевизору нужны только **Lampa** (интерфейс) и **Vimu** (плеер). На Шаге 9 вы просто укажете Lampa адрес вашего ПК.

### 📺 Samsung (Tizen) — через Media Station X
Downloader на Samsung **не работает** (Tizen — не Android). Поэтому для Samsung:
1. В магазине **Samsung Apps** установите **Media Station X**.
2. Запустите → **Settings → Start Parameter → Setup**.
3. Введите `lampa.mx`, подтвердите. Приложение перезапустится и откроет Lampa.

### 📺 LG (WebOS)
Через браузер [lampa.mx](https://lampa.mx) или доступные лаунчеры Lampa (см. сообщество WebOS).

> 📚 **Официальные гайды Lampa (от разработчиков):** [настройка с нуля](https://telegra.ph/Nastraivaem-Lampu-s-nulya-06-18) · [GitHub-wiki](https://github.com/yumata/lampa/wiki) · [установка на LG / Samsung / Hisense (Vidaa)](https://telegra.ph/USTANOVKA-i-zapusk-na-Lg-Samsung-Hisense-Vidaa-MCX-01-02) · [сообщество](https://t.me/lampa_group). Полный список ресурсов — в [техническом README](README.ru.md).

## Шаг 9. Подключить Lampa к серверу

В Lampa на телевизоре впишите адреса — **теперь с реальным IP вместо localhost**:

### TorrServer (воспроизведение)
**Настройки → Торренты** → включите TorrServer → основная ссылка:
```
http://ВАШ_АДРЕС:8090
```
Например: `http://192.168.1.165:8090`

### Jackett (поиск)
**Настройки → Парсер** → включите → впишите:
- **Ссылка:** `http://ВАШ_АДРЕС:9117` (напр. `http://192.168.1.165:9117`)
- **API ключ:** из файла `lampa_settings.txt` (или скопируйте из Jackett Dashboard, Шаг 4).

> 📌 Помните: **8090** — просмотр, **9117** — поиск. Не перепутайте порты.

## Шаг 10. Смотреть на ТВ 🍿

Найдите фильм → **«Смотреть»** → выберите раздачу с бо́льшим количеством **сидеров** → наслаждайтесь.

## Шаг 11. Автозапуск (чтобы не запускать вручную)

Чтобы сервер включался сам вместе с компьютером:
- **Docker Desktop → ⚙️ Settings → General →** галочка **«Start Docker Desktop when you log in»**.
- Контейнеры имеют `restart: unless-stopped`, так что поднимутся автоматически.

После перезагрузки ничего запускать не нужно: контейнеры поднимутся сами, а настройки TorrServer и Jackett сохранены и переживают рестарт. (При необходимости `configure.ps1` можно запустить повторно — он идемпотентный.)

## ❓ Если что-то не работает

| Проблема | Быстрое решение |
|---|---|
| Lampa на ТВ пишет «парсер не отвечает» | Проверьте адрес Jackett (`:9117`) и брандмауэр (Шаг 7). Подробно — FAQ №6 в [README](README.ru.md) |
| Видео не запускается | Проверьте адрес TorrServer (`:8090`) и что ТВ в той же сети |
| Нет звука | Выберите релиз **WEB-DL** вместо BDRemux, или внешний плеер (VLC/Vimu) — FAQ №1 |
| Медленно | Выберите торрент с бо́льшим количеством сидеров |
| Не видно субтитров | Выберите дорожку **Full**, а не **Forced** — FAQ в README |

## ⚙️ (Необязательно) Кеш и прекеш — панель TorrServer

У TorrServer есть своя панель: **[http://localhost:8090](http://localhost:8090)** (или `http://ВАШ_АДРЕС:8090` с другого устройства) → **Settings**. Установщик уже выставляет разумные значения, но при желании их можно подкрутить:

- **Cache (кеш)** — сколько видео держится в **оперативной памяти**. По умолчанию **2 ГБ**. Больше = плавнее на высоких битрейтах (4K), но и больше RAM. На слабых устройствах (Raspberry Pi, 2–4 ГБ RAM) уменьшите до 512 МБ – 1 ГБ.
- **Preload (прекеш)** — сколько процентов кеша загружается **заранее**, прежде чем начнётся воспроизведение. По умолчанию — **10%**: старт быстрый, а буфер набивается уже во время просмотра. Больший процент (напр. 50%) даёт более плавный старт на плохо раздаваемых торрентах, но придётся дольше ждать перед запуском и тратить больше RAM.

> 💡 Изменили — нажмите **Save**. Переустанавливать ничего не нужно: настройки сохраняются и переживают перезапуск. Значения по умолчанию задаёт установщик через `.env` (`TORRSERVER_CACHE_SIZE`, `TORRSERVER_PRELOAD`) — там их можно изменить навсегда.

## 🔒 Хочу скрыть свой IP-адрес

Проект умеет гонять торрент-трафик через бесплатный туннель **Cloudflare WARP**, чтобы другие участники видели не ваш адрес. Не обязательно, но доступно. Проще всего — включить WARP галочкой в установщике (см. «Самый лёгкий путь» в начале Части 1); детали — раздел «WARP» в [README](README.ru.md).
