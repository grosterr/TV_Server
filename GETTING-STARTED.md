# 🎬 Покрокова інструкція «з нуля»

Цей гайд проведе вас від порожнього комп'ютера до фільму на екрані — **навіть якщо ви ніколи не чули слова «Docker»**.

Гайд поділено на дві частини:

- **[ЧАСТИНА 1 — Використання на ПК](#частина-1--використання-на-пк)** — найпростіше. Дивитесь фільми на самому комп'ютері. Нічого не треба знати про мережу, IP-адреси й налаштування телевізора.
- **[ЧАСТИНА 2 — Як сервер для ТВ та Android](#частина-2--як-сервер-для-тв-та-android)** — продовження. Робите з комп'ютера домашній сервер, щоб дивитися на телевізорі, приставці чи телефоні.

> ⏱ Частина 1 — ~15 хвилин. Частина 2 — ще ~15 хвилин. Технічних знань не треба: вміти встановлювати програми й копіювати текст.

---

## 📖 Що це взагалі таке (простими словами)

Уявіть це як **власний Netflix, який показує фільми з торентів**, але без завантаження й без реклами. На комп'ютері крутиться маленький «двигун», який шукає та відтворює відео на льоту — нічого не зберігається на диск.

Три складові:
- **TorrServer** — «двигун», що перетворює торент на відеопотік.
- **Jackett** — «пошуковик» по торент-сайтах.
- **FlareSolverr** — помічник, що обходить захист Cloudflare на трекерах (щоб працювало більше сайтів).

Керуєте всім через застосунок **Lampa** — гарний каталог із постерами.

> ℹ️ **Важливо про Lampa (з офіційного FAQ розробників):** Lampa — це не готове рішення, а **«конструктор» (skeleton)**, який треба **самостійно налаштувати** (вказати ваш TorrServer і парсер — саме це ми й робимо нижче). Це окремий безкоштовний застосунок, доступний нативно майже скрізь:
> - **ПК:** [Lampa Desktop](https://github.com/Kolovatoff/lampa-desktop/releases) (Windows/Mac/Linux)
> - **Android / Android TV / приставки:** [офіційні релізи lampa-app/LAMPA](https://github.com/lampa-app/LAMPA/releases)
> - **Веб / Smart TV:** сайт [lampa.mx](https://lampa.mx)
> - **Вихідники / самохостинг:** [yumata/lampa](https://github.com/yumata/lampa)

# ЧАСТИНА 1 — Використання на ПК

> 🎯 Мета цієї частини: встановити все й дивитися фільми **на цьому ж комп'ютері**. Без мережевих налаштувань — усе через адресу `localhost` (тобто «цей комп'ютер»).

## Крок 1. Встановити Docker Desktop

«Docker» — програма, що запускає наш медіасервер в ізольованому «контейнері». Встановлюється як звичайний застосунок.

1. Відкрийте [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) → **Download for Windows**.
2. Запустіть файл, тисніть **OK / Next** до кінця, дозвольте перезавантаження за потреби.
3. Запустіть **Docker Desktop** (іконка кита 🐳). Дочекайтеся, поки кит унизу зліва стане зеленим — «готовий».

> ⚠️ Якщо Docker попросить полагодити «WSL 2» — він сам покаже кнопку. Натисніть її та перезавантажтесь.

## Крок 2. Завантажити цей проєкт

1. На сторінці проєкту на GitHub → зелена кнопка **`< > Code` → Download ZIP**.
2. Розпакуйте, наприклад, у `C:\media-server`.

## Крок 3. Перший запуск

1. Відкрийте підпапку **`media-server`**.
2. У порожньому місці всередині папки: **Shift + права кнопка миші** → **«Відкрити вікно PowerShell тут»**.
3. Введіть і натисніть **Enter**:
   ```powershell
   docker compose up -d
   ```
4. Перший раз кілька хвилин завантажуються компоненти. Коли побачите рядки `Started` — сервер працює. 🎉

**Перевірка:** відкрийте [http://localhost:9117](http://localhost:9117) — має відкритися Jackett.

## Крок 4. Налаштувати пошук (Jackett)

Тут ви кажете серверу, по яких сайтах шукати фільми. Усе робиться на панелі Jackett: **[http://localhost:9117/UI/Dashboard](http://localhost:9117/UI/Dashboard)**

1. **Скопіюйте API Key** — довгий рядок у правому верхньому куті. Він знадобиться далі.
2. **Додати трекер:** кнопка **`+ Add indexer`** → знайдіть потрібний (наприклад `Toloka`, `The Pirate Bay`, `1337x`) → натисніть жовтий **`+`**.
   - Публічні (PirateBay, 1337x) додаються одразу.
   - Приватні (Toloka) попросять ваш **логін і пароль** від того сайту.
   - 💪 Завдяки вбудованому FlareSolverr працюють навіть трекери із захистом Cloudflare (1337x тощо).
3. **Видалити/змінити трекер:** на тій самій панелі біля кожного доданого трекера є значки — кошик (видалити), гайковий ключ (налаштувати), лупа (перевірити).

> 💡 Можна автоматизувати додавання приватних трекерів і оптимізацію сервера скриптом `configure.ps1` — див. [технічний README](README.md).

## Крок 5. Дивитися на ПК 🍿

**Рекомендований спосіб — нативний застосунок [Lampa Desktop](https://github.com/Kolovatoff/lampa-desktop/releases)** (Windows/Mac/Linux). Він надійніший за браузер: пам'ятає налаштування, не має проблем із блокуванням «незахищених» запитів і легко під'єднує зовнішні плеєри.

1. Завантажте й встановіть Lampa Desktop із [релізів](https://github.com/Kolovatoff/lampa-desktop/releases).
2. У **Налаштуваннях** вкажіть два «локальні» адреси:
   - **TorrServer** (Налаштування → Торренти): `http://localhost:8090`
   - **Jackett / Парсер** (Налаштування → Парсер): посилання `http://localhost:9117` + **API Key** з Кроку 4.
3. Знайдіть фільм → **«Дивитись»** → оберіть роздачу (де більше **сідерів** — тим швидше) → відтворення за кілька секунд.

> 💡 **Швидка альтернатива без встановлення:** відкрийте [lampa.mx](https://lampa.mx) прямо в браузері й вкажіть ті самі адреси. Працює, але браузер може блокувати запити до `localhost` з HTTPS-сайту — тоді або погодьтеся на «небезпечний вміст» у налаштуваннях сайту, або скористайтеся десктоп-застосунком вище.
>
> Якщо відео не грає через формат — відкрийте потік у **VLC** («Відкрити у зовнішньому плеєрі») або через веб-інтерфейс TorrServer: [http://localhost:8090](http://localhost:8090).

**Готово! На комп'ютері все працює.** Якщо цього достатньо — можна зупинитися тут. Хочете дивитися на телевізорі — переходьте до Частини 2. 👇

# ЧАСТИНА 2 — Як сервер для ТВ та Android

> 🎯 Мета: перетворити комп'ютер на **домашній сервер**, щоб дивитися фільми на телевізорі, приставці чи телефоні. Комп'ютер робить усю роботу, інші пристрої лише показують.

**Що змінюється порівняно з Частиною 1:** замість `localhost` тепер треба вказувати **реальну адресу комп'ютера в мережі**, а також дозволити з'єднання у брандмауері.

### ⚠️ Головна умова
Комп'ютер і телевізор мають бути **в одній домашній мережі** (один роутер/Wi-Fi). І комп'ютер має бути **увімкнений під час перегляду**.

---

## Крок 6. Дізнатися адресу комп'ютера

1. У вікні PowerShell введіть:
   ```powershell
   ipconfig
   ```
2. Знайдіть рядок **«IPv4-адреса»** — щось на кшталт `192.168.1.165`.
3. Запишіть це число. **Це адреса вашого сервера** для всіх пристроїв.

> 💡 Щоб адреса не мінялася після перезавантаження, у налаштуваннях роутера «закріпіть» її за комп'ютером (Static DHCP / резервування). Не обов'язково, але зручно.

## Крок 7. Дозволити з'єднання у брандмауері

Windows за замовчуванням може блокувати вхідні підключення з інших пристроїв. Дозвольте порти сервера — у PowerShell **від імені адміністратора**:

```powershell
New-NetFirewallRule -DisplayName "MediaServer TorrServer" -Direction Inbound -LocalPort 8090 -Protocol TCP -Action Allow
New-NetFirewallRule -DisplayName "MediaServer Jackett"    -Direction Inbound -LocalPort 9117 -Protocol TCP -Action Allow
```

> Як відкрити PowerShell адміністратором: меню Пуск → напишіть «PowerShell» → правою кнопкою → «Запуск від імені адміністратора».

## Крок 8. Встановити Lampa на телевізор

### 📺 Samsung (Tizen)
1. У магазині **Samsung Apps** встановіть **Media Station X**.
2. Запустіть → **Settings → Start Parameter → Setup**.
3. Введіть `lampa.mx`, підтвердіть. Застосунок перезапуститься й відкриє Lampa.

### 🤖 Android TV / приставка / телефон
Встановіть офіційний застосунок **Lampa** з [релізів lampa-app/LAMPA](https://github.com/lampa-app/LAMPA/releases) (файл `.apk`). Раджу також поставити зовнішній плеєр (**Vimu** або **MX Player**) — знадобиться для деяких форматів звуку.

### 📺 LG (WebOS)
Через браузер [lampa.mx](https://lampa.mx) або доступні лаунчери Lampa (див. спільноту WebOS).

> 📚 Офіційні гайди та підтримка Lampa — у [Telegram-спільноті розробників](https://t.me/lampa_group) та на [GitHub](https://github.com/yumata/lampa).

## Крок 9. Під'єднати Lampa до сервера

У Lampa на телевізорі впишіть адреси — **тепер з реальним IP замість localhost**:

### TorrServer (відтворення)
**Налаштування → Торренти** → увімкніть TorrServer → основне посилання:
```
http://ВАША_АДРЕСА:8090
```
Наприклад: `http://192.168.1.165:8090`

### Jackett (пошук)
**Налаштування → Парсер** → увімкніть → впишіть:
- **Посилання:** `http://ВАША_АДРЕСА:9117` (напр. `http://192.168.1.165:9117`)
- **API ключ:** той самий рядок із Jackett Dashboard (Крок 4).

> 📌 Пам'ятайте: **8090** — перегляд, **9117** — пошук. Не переплутайте порти.

## Крок 10. Дивитися на ТВ 🍿

Знайдіть фільм → **«Дивитись»** → оберіть роздачу з більшою кількістю **сідерів** → насолоджуйтесь.

## Крок 11. Автозапуск (щоб не запускати вручну)

Щоб сервер вмикався сам разом із комп'ютером:
- **Docker Desktop → ⚙️ Settings → General →** галочка **«Start Docker Desktop when you log in»**.
- Контейнери мають `restart: unless-stopped`, тож підіймуться автоматично.

Після перезапуску комп'ютера один раз запустіть `configure.ps1` (або перевірте налаштування) — деталі в [README](README.md).

## ❓ Якщо щось не працює

| Проблема | Швидке рішення |
|---|---|
| Lampa на ТВ пише «парсер не відповідає» | Перевірте адресу Jackett (`:9117`) і брандмауер (Крок 7). Детально — FAQ №6 у [README](README.md) |
| Відео не запускається | Перевірте адресу TorrServer (`:8090`) і що ТВ у тій самій мережі |
| Немає звуку | Оберіть реліз **WEB-DL** замість BDRemux, або зовнішній плеєр (VLC/Vimu) — FAQ №1 |
| Повільно | Оберіть торент із більшою кількістю сідерів |
| Не видно субтитрів | Оберіть доріжку **Full**, а не **Forced** — FAQ у README |

📚 **Повний список рішень, оптимізація швидкості, приховування IP через WARP і таблиця якостей** — у [технічному README](README.md).

## 🔒 Хочу приховати свою IP-адресу

Проєкт уміє ганяти торент-трафік через безкоштовний тунель **Cloudflare WARP**, щоб інші учасники бачили не вашу адресу. Необов'язково, але доступно — розділ «WARP» у [README](README.md).
