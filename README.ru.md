Локальный медиасервер, разработанный для потокового воспроизведения торрентов и агрегированного поиска. Идеально подходит для использования с приложением **Lampa** на Smart TV, Android TV или ПК.

## 🚀 Архитектура и состав

Этот стек разворачивается в изолированной среде с помощью **Docker Compose** и состоит из следующих компонентов:

1. **TorrServer** (порт `8090`)
   - **Назначение:** Потоковое воспроизведение торрентов (Streaming) без необходимости предварительной загрузки всего файла на жёсткий диск.
   - **Оптимизация:** Скрипт `configure.ps1` включает расширенный кеш в оперативной памяти (2 ГБ) и увеличивает лимит P2P-соединений (до 1000) для максимальной утилизации ширины канала (до 300+ Мбит/с). Значения настраиваются в файле `.env`.
   - Данные кешируются исключительно в ОЗУ и удаляются после просмотра.

2. **Jackett** (порт `9117`)
   - **Назначение:** Унифицированный прокси-сервер для поиска по торрент-трекерам. Переводит поисковые запросы от Lampa в запросы к конкретным сайтам.
   - **Трекеры:** Поддерживает добавление любых открытых или закрытых трекеров (например: Toloka.to, ThePirateBay, 1337x).

3. **FlareSolverr** (порт `8191`)
   - **Назначение:** Обход Cloudflare-защиты («I'm not a robot», JS-челленджи) на трекерах. Без него Jackett не может индексировать защищённые сайты (напр. 1337x) — именно эта способность делает агрегаторы вроде JacRed «богаче на источники».
   - Jackett автоматически обращается к нему для проблемных трекеров; адрес задаётся в `configure.ps1` / `.env` (`JACKETT_FLARESOLVERR_URL`).

## 🛠 Как запустить (Установка)

### ⚡ Быстрый путь — установщик (рекомендуется)

Один скрипт с **меню-галочками** поднимет стек и сделает всю рутину: сгенерирует `.env`, **сам подхватит API-ключ Jackett** (без копирования), включит CORS и подключит FlareSolverr, настроит TorrServer, добавит выбранные трекеры и в конце напечатает готовые ссылки для Lampa.

- **Windows:** дважды кликните **`media-server/install.bat`**.
- **Linux / NAS / Raspberry Pi:**
  ```bash
  cd media-server && bash install.sh
  ```

В меню стрелками/пробелом выберите нужное (скрытие IP через WARP, приватные трекеры) — и всё. Скрипт **идемпотентный**, запускайте повторно когда угодно. Нужен лишь установленный Docker; проброс порта на роутере установщик напомнит, но автоматизировать не может.

<details>
<summary><b>Или вручную, пошагово ⤵️</b></summary>

1. **Подготовка:**
   Убедитесь, что у вас установлен [Docker Desktop](https://www.docker.com/products/docker-desktop/) (если вы на Windows) или Docker + Docker Compose (если на Linux).
   **Важно для автозапуска:** В настройках Docker Desktop (Settings -> General) включите галочку **"Start Docker Desktop when you log in"**. Благодаря параметру `restart: unless-stopped`, сервер будет автоматически запускаться вместе с ПК как фоновая служба.

2. **Запуск контейнеров:**
   Откройте терминал, перейдите в папку `media-server` и выполните команду:
   ```bash
   docker compose up -d
   ```

3. **Базовая настройка Jackett:**
   - Откройте браузер и перейдите по адресу: `http://localhost:9117`
   - В верхнем правом углу скопируйте ваш **API Key**. Он понадобится для подключения к Lampa.

4. **Автоматическая настройка (рекомендуется):**
   Вместо того чтобы добавлять каждый трекер вручную, воспользуйтесь скриптом. Он настроит оптимизацию TorrServer и добавит все приватные трекеры за один шаг.
   - Скопируйте шаблон и впишите свои данные:
     ```powershell
     Copy-Item .env.example .env   # затем отредактируйте .env
     ```
   - В `.env` укажите `JACKETT_APIKEY` (из веб-интерфейса Jackett) и логины/пароли трекеров.
   - Запустите:
     ```powershell
     ./configure.ps1
     ```
   Скрипт **идемпотентный** — его можно безопасно запускать повторно. Чтобы добавить новый приватный трекер, достаточно дописать одну строку в массив `$Trackers` внутри `configure.ps1` и соответствующие переменные в `.env`.

</details>

> 🔒 **Безопасность:** файл `.env` содержит пароли и API-ключ и **не попадает в git** (добавлен в `.gitignore`). Никогда не коммитьте его. Публичные трекеры (ThePirateBay, 1337x) не требуют логина — их можно добавить кнопкой **"+ Add indexer"** в веб-интерфейсе Jackett.

## 📺 Подключение к Lampa

> ℹ️ **О Lampa (официальный FAQ разработчиков):** Lampa — это не готовый продукт, а **«конструктор»**, который нужно самостоятельно настроить (указать ваш TorrServer и парсер). Официальные приложения: [Lampa Desktop](https://github.com/Kolovatoff/lampa-desktop/releases) (ПК), [lampa-app/LAMPA](https://github.com/lampa-app/LAMPA/releases) (Android/TV), [lampa.mx](https://lampa.mx) (веб), [yumata/lampa](https://github.com/yumata/lampa) (самохостинг). Сообщество: [t.me/lampa_group](https://t.me/lampa_group).

Для того чтобы Lampa увидела ваш новый сервер и начала искать и воспроизводить фильмы, выполните следующие шаги:

### Вариант А: Android TV / приставка — через Downloader
Самый удобный способ для Android TV: приложение **Downloader** ставит Lampa и плеер Vimu по одному коду.
1. Установите **Downloader by AFTVnews** из Google Play (оранжевый значок со стрелкой).
2. Запустите → введите код **`4384169`** → **Go**. Откроется страница «Загрузка APK».
3. Выберите и установите **Lampa** (обязательно) и **Vimu Media Player** (внешний плеер для DTS/4K). Torrserve MatriX ставить **не нужно** — сервер уже на вашем ПК.
4. При необходимости разрешите установку из неизвестных источников для Downloader (ТВ подскажет).

### Вариант Б: Samsung Tizen — через Media Station X
Downloader на Samsung не работает (Tizen — не Android), поэтому:
1. Загрузите **Media Station X** из официального магазина Samsung Apps.
2. Запустите его ➔ **Settings** ➔ **Start Parameter** ➔ **Setup**.
3. Введите `lampa.mx` и подтвердите. Приложение перезапустится и откроет Lampa.

### Вариант В: Настройка самой Lampa (все устройства)
1. **TorrServer:** Перейдите в **Настройки** -> **Торренты**.
   - Включите использование TorrServer.
   - Укажите **Основную ссылку**: `http://[IP-АДРЕС-ВАШЕГО-ПК]:8090` (например, `http://192.168.1.165:8090`).
2. **Jackett:** Перейдите в **Настройки** -> **Плагины**.
   - Перейдите в настройки плагина Jackett и укажите:
   - **Ссылка:** `http://[IP-АДРЕС-ВАШЕГО-ПК]:9117`
   - **API ключ:** Вставьте скопированный ключ из веб-интерфейса Jackett.

## ⚡ Оптимизация скорости на P2P-уровне

Если скорость низкая даже на торрентах с большим количеством сидеров — самая частая причина в том, что ваш TorrServer **недоступен для входящих соединений** (за NAT). Тогда вы можете лишь сами подключаться к открытым пирам, а к вам — никто. Это режет скорость в разы.

**Решение — фиксированный порт для пиров + проброс на роутере:**

1. **Порт уже настроен** в этом репозитории: `42116` (TCP+UDP) открыт в `docker-compose.yml` и задан в TorrServer (`PeersListenPort`) через `configure.ps1`. Изменить можно в `.env` (`TORRSERVER_PEER_PORT`).

2. **Проброс на роутере (обязательно для WAN).** Зайдите в настройки роутера → **Port Forwarding / Виртуальный сервер** и добавьте правило:
   - Внешний порт: `42116`, протокол: **TCP и UDP**
   - Внутренний IP: IP вашего ПК (напр. `192.168.1.165`), внутренний порт: `42116`

3. **Брандмауэр Windows.** Разрешите входящие на `42116` (единоразово, в PowerShell от администратора):
   ```powershell
   New-NetFirewallRule -DisplayName "TorrServer P2P" -Direction Inbound `
     -LocalPort 42116 -Protocol TCP -Action Allow
   New-NetFirewallRule -DisplayName "TorrServer P2P UDP" -Direction Inbound `
     -LocalPort 42116 -Protocol UDP -Action Allow
   ```

4. **Перезапуск при изменении порта:** новые порты требуют пересоздания контейнера, не просто рестарта:
   ```powershell
   docker compose up -d torrserver
   ```

**Другие рычаги (при необходимости):**
- **Провайдер режет торренты?** Некоторые провайдеры замедляют P2P. В веб-интерфейсе TorrServer можно включить шифрование протокола (`ForceEncrypt`) — иногда обходит троттлинг (но немного уменьшает пул пиров).
- **DHT / PEX / uTP** уже включены — они дают дополнительных пиров сверх списка трекера. Не выключайте.
- **Главное, что не лечится настройками:** количество сидеров на торренте. Выбирайте релизы с большим S — это определяет потолок скорости (см. FAQ).

> Проверить, что проброс работает: в веб-интерфейсе TorrServer во время загрузки должно расти количество *активных пиров*, а не только *pending*.

### Скрыть IP через Cloudflare WARP (без платного VPN)

Если вы за «серым» IP (CGNAT) или просто хотите, чтобы пиры видели не ваш IP — весь трафик TorrServer можно завернуть через бесплатный **Cloudflare WARP** (сервис `warp` на базе [gluetun](https://github.com/qdm12/gluetun) в `docker-compose.warp.yml`). Проще всего — выбрать WARP в установщике; ниже — ручной способ. Проверено: P2P через туннель работает, скорость на хорошо раздаваемых торрентах не страдает.

**Что даёт / чего не даёт:**
- ✅ Пиры видят IP Cloudflare, а не ваш.
- ✅ Может обойти троттлинг BitTorrent со стороны провайдера.
- ❌ **Не** ускоряет само по себе и **не** даёт входящего порта (вы за NAT Cloudflare) — потолок скорости определяет количество сидеров.
- ⚠️ Cloudflare может резать P2P — если скорость упадёт до нуля, туннель стоит выключить.

**Настройка (единоразово):**
1. Сгенерируйте WARP-профиль (создаёт бесплатный аккаунт):
   ```powershell
   docker run --rm -v "${PWD}\warp:/data" -w //data alpine sh -c `
     "apk add --no-cache curl >/dev/null && curl -sL -o wgcf https://github.com/ViRb3/wgcf/releases/download/v2.2.22/wgcf_2.2.22_linux_amd64 && chmod +x wgcf && ./wgcf register --accept-tos && ./wgcf generate && cat wgcf-profile.conf"
   ```
2. Из вывода возьмите `PrivateKey` и первый `Address` и впишите в `.env`:
   ```
   WARP_PRIVATE_KEY=<PrivateKey>
   WARP_ADDRESS_V4=172.16.0.2/32
   ```
3. Поднимите стек с туннелем: `docker compose -f docker-compose.warp.yml up -d`
4. Проверьте, что IP изменился на Cloudflare: `docker exec warp wget -qO- https://api.ipify.org`

**Выключить туннель:** просто запускайте обычный стек — `docker compose up -d` (файл `docker-compose.yml` уже без WARP, TorrServer публикует порты напрямую). WARP включается только отдельным файлом `docker-compose.warp.yml`.

> Папка `warp/` (содержит ключи аккаунта) и `.env` — в `.gitignore`, в git не попадают.

## 🎬 Битрейты и нужная скорость (720p → 4K)

Битрейт — это сколько данных в секунду содержит видео. Для стриминга через TorrServer он означает одно: **какую скорость загрузки должны обеспечить сидеры, чтобы фильм не буферизировался**. Если битрейт релиза выше того, что дают сидеры — будет подгрузка, каким бы быстрым ни был ваш интернет.

| Качество | Кодек | Оптимальный битрейт | Нужно ↓ | Размер (~2 ч) |
|---|---|---|---|---|
| **720p** | H.265 (HEVC) | ~4 Мбит/с (3–6) | ~0.5 МБ/с | ~4 ГБ |
| **720p** | H.264 | ~6 Мбит/с (4–8) | ~0.75 МБ/с | ~5 ГБ |
| **1080p** | H.265 (HEVC) | ~8 Мбит/с (6–12) | ~1 МБ/с | ~7 ГБ |
| **1080p** WEB-DL | H.264 | ~12 Мбит/с (8–15) | ~1.5 МБ/с | ~11 ГБ |
| **1080p** BluRay | H.264 | ~28 Мбит/с (20–40) | ~3.5 МБ/с | ~22 ГБ |
| **1440p (2K)** | H.265 (HEVC) | ~16 Мбит/с (12–24) | ~2 МБ/с | ~14 ГБ |
| **4K (2160p)** WEB-DL | HEVC | ~35 Мбит/с (25–50) | ~4.5 МБ/с | ~30 ГБ |
| **4K (2160p)** BluRay/UHD | HEVC | ~65 Мбит/с (50–90) | ~8 МБ/с | ~55 ГБ |
| **4K Remux** | HEVC (без сжатия) | ~100 Мбит/с (80–120+) | ~12.5 МБ/с | ~80 ГБ |

**Как читать таблицу:**
- Значения — суммарный битрейт (видео + аудио), ориентировочный. HDR, 60 fps и динамичные сцены смещают к верхней границе.
- **«Нужно ↓»** — это минимальная стабильная скорость загрузки. Вам нужны **сидеры, способные её держать**, а не только быстрый тариф. Например, 4K Remux требует ~12 МБ/с — если на раздаче 2 сидера по 0.1 МБ/с, он будет тормозить даже на канале 300 Мбит/с.
- **H.265 (HEVC)** даёт то же качество вдвое меньшим битрейтом, чем H.264 — но старые/слабые устройства могут его не тянуть.
- **Золотая середина для Smart TV (Samsung/LG):** **1080p WEB-DL H.264** — умеренный битрейт, совместимый звук (AAC), много сидеров. 4K Remux бери только когда сидеров сотни и плеер точно тянет формат.

## 🔍 Мониторинг и логи (PowerShell)

Все команды работают из любой папки — Docker находит контейнеры по имени (`jackett`, `torrserver`).

### Логи Jackett (поиск / парсер)
```powershell
docker logs jackett --tail 50          # последние 50 строк
docker logs jackett -f                 # вживую, в реальном времени (Ctrl+C — выйти)
docker logs jackett --since 10m        # только за последние 10 минут
docker logs jackett -f --timestamps    # вживую + время каждой строки
```

### Логи TorrServer (воспроизведение / загрузка)
```powershell
docker logs torrserver --tail 50
docker logs torrserver -f
```

### Фильтровать логи (аналог grep)
```powershell
docker logs jackett --tail 200 | Select-String "Error"            # только ошибки
docker logs jackett --tail 200 | Select-String "search"           # только поиск
docker logs jackett --tail 200 | Select-String "Error","Found"    # несколько слов
```

### Скорость и сидеры торрента без веб-интерфейса
```powershell
# список активных торрентов + их хеши
Invoke-RestMethod -Uri "http://127.0.0.1:8090/torrents" -Method Post `
  -ContentType 'application/json' -Body '{"action":"list"}' | Select-Object title, hash

# скорость/сидеры конкретного торрента (подставь свой hash из списка выше)
$h = "ПОДСТАВЬ_СВОЙ_HASH"
$t = Invoke-RestMethod -Uri "http://127.0.0.1:8090/torrents" -Method Post `
  -ContentType 'application/json' -Body (@{action='get';hash=$h}|ConvertTo-Json)
"{0} MB/s ↓ | {1} сидеров | {2} пиров" -f `
  [math]::Round($t.download_speed/1MB,2), $t.connected_seeders, $t.active_peers
```

### Шпаргалка
| Хочу | Команда |
|---|---|
| Свежие логи | `docker logs jackett --tail 50` |
| Смотреть вживую | `docker logs jackett -f` |
| Что работает сейчас | `docker ps` |
| Перезапустить | `docker restart jackett` |

> 💡 Удобнее всего для диагностики ТВ: держи открытым `docker logs jackett -f` и сделай поиск на телевизоре — сразу видно, прилетает ли запрос и что Jackett отвечает.

## 💡 Частые вопросы (FAQ) и решение проблем

### 1. Нет звука во время просмотра (BDRip / Blu-Ray)
- **Причина:** Большие торренты в качестве BDRip/Remux используют многоканальные аудиоформаты (**AC3 / DTS**). Встроенные плееры в браузерах, WebOS и Tizen часто не имеют лицензии на их воспроизведение, поэтому видео идёт без звука.
- **Решение:**
  - На ПК: Используйте внешний плеер (VLC, PotPlayer), выбрав в Lampa опцию "Открыть в VLC".
  - На Smart TV (Android): Установите Vimu Media Player, MX Player или VLC и выберите его в настройках Lampa (Настройки -> Плеер).
  - На телевизорах Samsung/LG: Выбирайте для просмотра торренты формата **WEB-DL** / **WEBRip** (у них обычно совместимый звук формата AAC).

### 2. Почему скорость отображается как 30 МБ/с, если у меня тариф 300 Мбит/с?
- Обратите внимание на единицы измерения: провайдеры указывают скорость в Мега**битах** (Мбит/с), а Lampa или TorrServer часто показывают в Мега**байтах** (МБ/с, МБайт/с).
- Поскольку 1 Байт = 8 бит, скорость 33 МБ/с на самом деле равна ~265 Мбит/с. Это означает, что программа работает на максимуме!

### 3. Скорость загрузки не достигает максимума, а "прыгает"
- Торрент (BitTorrent) — это P2P-сеть. Скорость зависит от количества обычных людей (сидеров), которые сейчас раздают этот файл, и их скорости отдачи.
- Также TorrServer "умно" снижает скорость, когда кеш заполняется, чтобы не перегружать компьютер. Фильм не загружается полностью, а лишь буферизируется на лету.

### 4. Что будет с фильмом, если перезагрузить компьютер?
- Если вы перезагрузите ПК (на котором работает сервер) во время просмотра, фильм на телевизоре зависнет. Поскольку TorrServer кеширует видео только в оперативной памяти (чтобы не изнашивать жёсткий диск), после рестарта ПК кеш очистится.
- **Как продолжить:** Просто включите фильм на телевизоре снова и перемотайте на нужную минуту. TorrServer мгновенно подключится к сидерам и продолжит загрузку именно с того момента (обычно это занимает 3-5 секунд).

### 5. Фильм внезапно зависает, а после переподключения вместо названия — длинный hash
- **Причина:** TorrServer периодически падал из-за дефекта в самом TorrServer MatriX.142 (открытый баг [YouROK/TorrServer#766](https://github.com/YouROK/TorrServer/issues/766) — гонка данных в torrent-библиотеке под параллельной нагрузкой). Docker автоматически перезапускает сервис за ~5 секунд; пока Lampa заново тянет метаданные торрента, она показывает его infohash — это временно.
- **Решение (уже применено в этом репозитории):** версия TorrServer закреплена на стабильной `MatriX.141.10` через образ `ghcr.io/yourok/torrserver` (тег задаётся переменной `TORRSERVER_VERSION` в `.env`). Образ `yourok/torrserver:latest` **не используйте** — он при каждом старте загружает новейший бинарник с GitHub, то есть версия неконтролируемая.
- Когда баг #766 исправят — просто поднимите `TORRSERVER_VERSION` в `.env` и выполните `docker compose up -d torrserver`.
- Проверить текущую версию: `Invoke-RestMethod http://127.0.0.1:8090/echo`. Данные/настройки теперь хранятся в `torrserver_data/` и переживают пересоздание контейнера.

### 6. Lampa на телевизоре пишет «парсер не отвечает», хотя с телефона страница `:9117` открывается
Это почти всегда одна из двух причин (страница открывается с телефона, потому что это обычная навигация браузера, а Lampa делает фоновые JS-запросы, к которым применяются другие правила безопасности):

1. **CORS выключен в Jackett.** Браузер блокирует ответ Jackett, потому что запрос идёт с чужого домена (`lampa.mx`).
   - **Решение:** в файле `jackett_config/Jackett/ServerConfig.json` установите `"AllowCORS": true` и перезапустите контейнер: `docker restart jackett`.
2. **Mixed content (HTTPS → HTTP).** Если Lampa открыта по `https://`, а Jackett работает по `http://`, браузер телевизора блокирует «незащищённый» запрос.
   - **Решение:** в Media Station X в поле *Start Parameter* укажите Lampa через **http** (`http://lampa.mx`), чтобы и Lampa, и Jackett были по HTTP.

Также проверьте, что:
- В Lampa указан именно LAN-IP ПК, а не `localhost`: `http://192.168.1.165:9117` (свой IP: `ipconfig` → строка IPv4).
- Телевизор и ПК в **одной сети** (без «гостевого» Wi-Fi или AP-isolation).
- Брандмауэр Windows пропускает входящие на порт `9117` для приватной сети.
